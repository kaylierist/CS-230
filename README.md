# CS-230 
The client of the Gaming Room application was the developers of the Gaming Room, a game similar to Draw It or Lose It, who wanted their game to be available via an interface that could extend to multiple operating systems. I think I considered the benefits and constraints of how to go about dveloping this application well. It was helpful to be given specific attributes desired for the game, and to be given multiple UML diagrams mapping out how the game was to look in its final stages. If I could revise a part of this project it would be the code itself, because I think I could have made it more simplified and also better working if I had given myself more time and paid better attention to detail. The design of the software had to change a few times based on the user's needs, which is the most important part of a project that is client-driven, because meeting their requirements for the software is the main job of the developers. In the future, I would continue to use the strategies of mapping out what I think a project should look like and getting comfortable with what I am doing, then taking the specifics from the client and ensuring all user needs are met. Creating a reliable and easy-to-use application takes a lot of consideration on multiple fronts, but a good team who covers all their bases and knows how to use an operating system to its full potential can create a worthwhile product that is stable and secure.
